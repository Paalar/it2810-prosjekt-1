Flexbox & CSS-Grid

Flexbox layout and CSS-grid are meant to solve the issues of aligning items on a webpage. The problems arises often when items are supposed to be dynamic and change to different screen sizes for all kind of screens be it phone or computer screens.

With Flexbox a lot of the items are generated automatically, but can be set to how much of the screen the item should use. With CSS grid and how many columns are manually set, while each item is either given or not how many columns it should span or in which column it should be placed. With Flexbox it was a lot easier to make the site pleasing and dynamic which is why Saran chose to use it over CSS-Grid.

SVG & Canvas

SVG & Canvas are both used to draw graphical elements on a webpage in html. SVGs are drawn in the html's svg tag while Canvas uses the canvas tag, but is draw in a script. Canvas is more suited for games, but the main reason Saran chose SVGs is the support for event handlers as well as they are resolution independent and they take less space than canvas. It can be argued that Canvas would be well suited since the art uses animations, but Saran favoured the other advantages.

JQuery

JQuery is a lightweight JavaScript library which requires less writing than pure JavaScript. JQuery takes a lot of functionality in JavaScript that requires a lot of lines to write which can easily be done in a single or few methods in JQuery. JQuery simplifies DOM manipulation and create event handlers like on clikc functions as well as easily edit css.

In this project the use of JQuery makes it easy to select and save either classes or IDs into a single variable. Several SVG elements are created to look like buttons, and it is easy to create an on Click function, with only the .click(); method, shown with an example: variable.click( function (){ //code });. The possibility to edit css is easily made by calling the method .css() on a variable, shown with an example: variable.css('attribute', 'value'); The final and also useful function used with JQuery is the .is(); method called on a variable. It was used to check if an element was visible, shown with an example variable.is(':visible');

Cross browser testing

Cross browser testing is incredibly important in creating websites since there exists so many different devices and browsers used by so many people in different parts of the world. Not only is it important to test different browsers, but also different browser versions where a lot can change in a few years time. Screens may also vary in sizes, it isn't only computers using websites, but also phones and other smaller devices. So testing if the website is functional and looks good on different screens is also an important part of web devlopment.

Testing som har blitt gjort er på Mac og PC i både de nyeste versjonene av Firefox og Google Chrome. Det ble testet i ulike skjerm størrelser for å se at Flexbox reagerte på ønskede måter. Det ble sjekket at alle animasjoner fungerte.
